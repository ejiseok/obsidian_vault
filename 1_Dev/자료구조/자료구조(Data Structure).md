
# 개념

- 컴퓨터가 데이터를 효율적으로 다룰 수 있게 도와주는 **데이터 보관 방법과 데이터에 관한 연산의 총체**를 뜻한다.
- 예를 들어, 프로그래밍 언어의 `int`도 자료구조
	- 32비트 메모리 공간 안에 수를 할당하되 첫 비트를 부호 표현에 사용하는 등의 '보관 방법'을 정의
	- 덧셈/뺄셈/나눗셈/곱셉/논리/시프트 등 다양한 '연산' 또한 정의하고 있다.

# 자료구조가 중요한 이유

1. 라이브러리에서 엉뚱한 자료구조를 선택하는 일을 피할 수 있다.
	- 동일한 [[ADT(Abstract Data Types)|ADT]]를 사용하더라도 자료구조에 따라 애플리케이션 성능이 크게 달라질 수 있기 때문
2. [[알고리즘(Algorithm)|알고리즘]]이 데이터를 효율적으로 사용할 수 있게 도와주는 핵심 부품 역할을 하기 때문이다.

# 종류

## 단순 자료구조(Primitive Data Structure)

- 프로그래밍 언어에서 통상적으로 제공하는 기본 데이터 형식을 말한다.
- `int`
- `long`
- `double`
- ...

## 복합 자료구조(Non-Primitive Data Structure)

- 복합 자료구조는 다시 선형 자료구조(Linear Data Structure)와 비선형 자료구조(Non-Linear Data Structure)로 나뉜다.
